# Research Discovery and Dissemination

The repository is structured for indexing by GitHub, Google, Bing, scholarly
software catalogs, and memory-system researchers.

## Automated Surfaces

- GitHub Pages research index with `SoftwareSourceCode` and `Dataset` JSON-LD
- `sitemap.xml` and `robots.txt`
- GitHub topics for memory benchmarks and evaluated datasets
- `CITATION.cff` and `codemeta.json`
- Stable JSON receipts and Markdown comparison tables
- Daily GitHub Actions artifacts

## Publication Checklist

1. Run WizeMe and competitors on the same pinned public dataset revision.
2. Merge normalized receipts and regenerate `comparison/latest.md`.
3. Create a GitHub release and archive it with Zenodo for a DOI.
4. Submit the repository to Papers with Code or the relevant benchmark registry.
5. Publish a methods post that links raw receipts, hardware, commands, and commit.
6. Share the release with the LoCoMo, LongMemEval, and PersonaMem communities.
7. Product Hunt promotion is pending external launch proof: load the live
   Product Hunt URL and screenshot receipt before marking launch validation
   complete.

## Suggested Paper Title

“WizeMe Benchmarks: Provenance-First Evaluation of Retrieval, Latency, and
Forgetting in Long-Term Conversational Memory”

Avoid “perfect memory” in the title until public benchmark receipts support it.

## Suggested Article Title

“Comparing Long-Term AI Memory Systems on LoCoMo, LongMemEval, PersonaMem, and
Forgetting-Aware Accuracy”

This title is discoverable without claiming results that have not been run.
